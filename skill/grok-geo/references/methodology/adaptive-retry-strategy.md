# 自适应重试策略

## 目标

当搜索失败或结果质量低时，**自动选择最优重试策略**，而非简单重复相同查询。

## 失败分类

### 硬失败（Hard Failures）— 不重试

| 错误类型 | 错误码 | 处理 |
|----------|--------|------|
| 搜索工具不可用 | `WEB_SEARCH_UNAVAILABLE` | 标记失败，跳过 |
| 被拒答（安全/合规） | `REFUSED` | 标记失败，记录原因 |
| 搜索工具超时（多次） | `TOOL_TIMEOUT` | 标记失败 |

### 软失败（Soft Failures）— 可重试

| 错误类型 | 错误码 | 重试策略 |
|----------|--------|----------|
| 搜索未实际使用 | `WEB_SEARCH_NOT_USED` | 策略 A：强化 prompt |
| 空回答 | `EMPTY_ANSWER` | 策略 B：查询改写 |
| 速率限制 | `RATE_LIMITED` | 策略 C：延迟重试 |
| 工具输出无效 | `INVALID_TOOL_OUTPUT` | 策略 D：简化请求 |

### 低质量结果（Low Quality）— 可优化重试

| 质量问题 | 检测方式 | 重试策略 |
|----------|----------|----------|
| 回答过短（<50 字） | `response_length < 50` | 策略 B：查询扩展 |
| 无引用来源 | `citation_count == 0` | 策略 E：指定来源要求 |
| 纯模型记忆（无搜索痕迹） | 无引用 + 无来源卡片 | 策略 A：强化搜索要求 |
| 拒答/回避 | `is_refusal == true` | 策略 F：换角度提问 |
| 品牌未提及但应提及 | 高价值问题 + 品牌未出现 | 策略 G：补充上下文 |

## 重试策略

### 策略 A：强化 Prompt

当搜索未实际执行时，在 prompt 中明确要求使用搜索工具。

```text
原始："{question}"
重试："请使用搜索工具回答以下问题：{question}。必须基于搜索结果给出回答。"
```

### 策略 B：查询改写

当回答为空或过短时，改写查询使其更具体。

```text
原始："适合50人团队的项目管理软件有哪些？"
重试："2026年适合50人研发团队使用的项目管理软件推荐对比"
```

**改写规则：**
- 添加年份（2026）
- 添加"推荐"或"对比"
- 扩展场景描述
- 不改变核心意图

### 策略 C：延迟重试

当遇到速率限制时，等待后重试。

```text
等待时间：5 秒 → 10 秒 → 20 秒（指数退避）
最大重试：3 次
```

### 策略 D：简化请求

当工具输出无效时，简化请求复杂度。

```text
原始：多问题批量请求
重试：单问题单次请求
```

### 策略 E：指定来源要求

当无引用来源时，要求引用具体来源。

```text
原始："{question}"
重试："回答以下问题并引用具体来源：{question}。请提供至少2个引用链接。"
```

### 策略 F：换角度提问

当被拒答或回避时，换一个角度提问。

```text
原始："示例科技和竞品A哪个好？"
重试："项目管理软件选购时需要考虑哪些因素？示例科技和竞品A分别在这些因素上表现如何？"
```

### 策略 G：补充上下文

当品牌未出现在高价值问题中时，补充行业上下文。

```text
原始："适合50人团队的项目管理软件有哪些？"
重试："在项目管理软件品类中，适合50人研发团队的工具有哪些？请覆盖主流和新兴产品。"
```

## 重试决策树

```
搜索结果返回
    │
    ├─ 硬失败？ → 标记失败，不重试
    │
    ├─ 软失败？
    │   ├─ WEB_SEARCH_NOT_USED → 策略 A
    │   ├─ EMPTY_ANSWER → 策略 B
    │   ├─ RATE_LIMITED → 策略 C
    │   └─ INVALID_TOOL_OUTPUT → 策略 D
    │
    ├─ 低质量？
    │   ├─ 无引用 → 策略 E
    │   ├─ 拒答 → 策略 F
    │   └─ 品牌未出现（高价值）→ 策略 G
    │
    └─ 正常 → 保存结果，不重试
```

## 重试限制

| 约束 | 值 | 原因 |
|------|---|------|
| 每题最大重试次数 | 1 | 避免浪费搜索配额 |
| 总重试比例 | ≤ 20% | 避免整体搜索时间膨胀 |
| 延迟重试最大等待 | 20 秒 | 避免过长等待 |
| 重试后仍失败 | 标记 `retry_failed` | 不阻断流程 |

## Agent 使用指南

1. 每次搜索结果返回后，检查结果质量
2. 按决策树选择重试策略（如有）
3. 最多重试 1 次
4. 记录重试前后对比
5. 重试结果覆盖原始结果（保留原始记录在 `retry.original_attempt`）
6. 统计重试成功率写入 `metrics.json` → `retry_stats`
