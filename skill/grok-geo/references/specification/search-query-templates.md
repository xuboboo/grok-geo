# 搜索查询模板库

## 目标

为每种问题意图（intent）提供**标准化的查询模板**，确保搜索覆盖品牌可见度的关键场景，避免遗漏高价值查询。

## 5 大查询模板类型

### 1. Best Tools（最佳推荐）

**意图**：用户寻找品类中的最佳选择

```text
模板："{industry} 最好的 {product_type} 有哪些？"
示例："项目管理软件最好的有哪些？"
变体："2026 年值得推荐的 {product_type}"
```

**覆盖场景**：
- 品类认知（用户刚开始了解品类）
- 推荐列表（AI 生成"Top 10"类回答）
- 品牌首次进入用户视野

**品牌可见度关键指标**：是否出现在推荐列表中

### 2. Alternatives（替代方案）

**意图**：用户已有目标产品，寻找替代品

```text
模板："{competitor} 的替代品有哪些？"
示例："Jira 的替代品有哪些？"
变体："有没有比 {competitor} 更好的选择？"
```

**覆盖场景**：
- 竞品用户流失场景
- 价格/功能不满导致的替代需求
- 品牌作为竞品替代品被发现

**品牌可见度关键指标**：是否被列为竞品的替代方案

### 3. Recommendations（场景推荐）

**意图**：用户有具体场景，寻求针对性推荐

```text
模板："适合 {scenario} 的 {product_type} 推荐"
示例："适合 50 人研发团队的项目管理软件推荐"
变体："{persona} 该用什么 {product_type}？"
```

**覆盖场景**：
- 细分场景匹配
- 规模/行业/角色特定需求
- 高转化率场景

**品牌可见度关键指标**：是否在特定场景下被推荐

### 4. Reviews（评价对比）

**意图**：用户已知品牌，寻求评价和对比

```text
模板："{brand} 怎么样？好用吗？"
示例："示例科技的项目管理软件怎么样？"
变体："{brand} vs {competitor} 哪个好？"
```

**覆盖场景**：
- 品牌已进入用户考虑集
- 决策前的深度调研
- 正面/负面评价影响

**品牌可见度关键指标**：评价是否正面，是否被推荐

### 5. Comparisons（功能对比）

**意图**：用户在多个选项间做对比

```text
模板："{brand} 和 {competitor} 有什么区别？"
示例："示例科技和竞争品牌A有什么区别？"
变体："{brand} vs {competitor} 功能对比"
```

**覆盖场景**：
- 最终决策阶段
- 功能/价格/服务对比
- 品牌差异化展示

**品牌可见度关键指标**：在对比中是否被正面评价

## 问题生成与模板映射

### 标准模式（30 问题）

| 模板类型 | 数量 | 占比 | 说明 |
|----------|------|------|------|
| best_tools | 6 | 20% | 品类认知 + 推荐 |
| alternatives | 5 | 17% | 竞品替代 |
| recommendations | 8 | 27% | 场景推荐（最高价值） |
| reviews | 6 | 20% | 品牌评价 |
| comparisons | 5 | 17% | 功能对比 |

### Quick 模式（10 问题）

| 模板类型 | 数量 |
|----------|------|
| best_tools | 2 |
| alternatives | 2 |
| recommendations | 3 |
| reviews | 2 |
| comparisons | 1 |

## 查询生成约束

### 必须满足

1. **≥ 70% 问题不含品牌名**：避免搜索偏见
2. **≥ 30% 为推荐/对比类**：覆盖高转化场景
3. **品牌事实类 ≤ 20%**：避免过度关注品牌自身
4. **使用目标地区语言**：匹配目标市场
5. **符合目标用户画像**：使用用户会使用的表达

### 禁止

1. 不生成诱导品牌出现的问题（如"最好的项目管理软件是哪个？"暗示有唯一答案）
2. 不把品牌卖点直接写入非品牌问题
3. 不生成与目标用户无关的问题
4. 不重复已生成的意图

## 查询质量评估

每个生成的查询应通过以下检查：

```json
{
  "query": "适合50人研发团队的项目管理软件有哪些？",
  "quality_checks": {
    "no_brand_bias": true,
    "natural_language": true,
    "specific_scenario": true,
    "matches_target_persona": true,
    "covers_high_value_intent": true,
    "not_duplicate": true
  },
  "quality_score": 6,
  "max_score": 6
}
```

## Agent 使用指南

1. Phase 3（Question Map Generation）时，按模板类型分配问题数量
2. 每个模板类型生成对应数量的问题
3. 问题生成后通过质量评估检查
4. 不合格问题重新生成
5. 最终问题列表写入 `intermediate/questions.json`，每个问题附带 `template_type` 字段
