# 推荐判定规则

## 计为推荐的情况

1. **首选/Top 推荐**：明确表示"首选"、"第一选择"、"最推荐"
2. **优先建议**：使用"优先"、"建议"、"适合"等正向建议词
3. **推荐列表**：出现在有排序的推荐列表中（如"5个最佳XX"）
4. **明确候选**：被列为"值得考虑"、"不错的选择"等
5. **有条件正向**：在特定条件下给出正向建议（如"如果你是小团队，推荐X"）

## 不计为推荐的情况

1. **市场背景**：仅作为行业背景或市场描述提及
2. **引用标题**：仅在引用来源标题中出现
3. **普通列举**：在无排序的长列表中简单罗列
4. **只谈缺点**：内容主要讨论不足或问题
5. **明确不推荐**：直接表达不建议使用
6. **公司历史**：仅出现在企业历史或融资信息中
7. **问题无关**：出现在与问题无关的上下文中

## 推荐强度

| 强度 | 信号词 | mention_type |
| --- | --- | --- |
| 强 | "最佳"、"首选"、"第一"、"最推荐" | strongly_recommended |
| 中 | "推荐"、"适合"、"不错"、"值得" | recommended |
| 弱 | "也可以"、"能用"、"作为备选" | weakly_recommended |

## 推荐排名

- 仅当搜索结果中出现明确的有序列表时才记录 recommendation_rank
- rank=1 表示第一个被推荐
- 无序列表不记录排名
- 不得仅凭出现顺序推断排名
